# Flow

A native macOS application starter template built with SwiftUI, with built-in code quality and security tooling.

## Features

- **SwiftUI** native macOS application (macOS 14.0+)
- **XcodeGen** for diff-friendly, generated Xcode projects
- **Pre-commit hooks** enforcing code quality, test coverage, and security scanning
- **SwiftLint** for consistent Swift code style (strict mode, with complexity/length limits)
- **Semgrep** SAST scanning for security vulnerabilities
- **gitleaks** secret detection to prevent credential leaks
- **Code coverage** thresholds enforced on every commit (75% per file)

## Requirements

- **macOS 14.0+**
- **Xcode 16+** — install from the App Store or [developer.apple.com](https://developer.apple.com/xcode/)
- **Homebrew** — install from [brew.sh](https://brew.sh)

After installing Xcode, you must accept the license and install components:

```bash
sudo xcodebuild -license accept
xcodebuild -runFirstLaunch
```

## Quick Start

```bash
# Clone and setup
git clone https://github.com/LukeFerris/Agentic-base-MacOS.git flow
cd flow
make setup

# Open in Xcode
make open

# Or build and test from command line
make build
make test
```

`make setup` installs all remaining dependencies (XcodeGen, SwiftLint, Semgrep, gitleaks), configures the pre-commit hook, and generates the Xcode project.

```
│   └── Info.plist           # App metadata

project.yml                  # XcodeGen project definition
.swiftlint.yml               # SwiftLint configuration
Makefile                     # Common commands
CLAUDE.md                    # AI agent working guidelines
```

> **Note:** `Flow.xcodeproj` is generated by XcodeGen from `project.yml` and is gitignored. Run `make generate` to recreate it after changes to `project.yml`.

## Pre-commit Checks

Every commit automatically runs a 6-step pipeline:

1. **Secret scanning** (gitleaks) — prevents committing credentials
2. **SAST scanning** (semgrep) — catches security vulnerabilities
3. **SwiftLint** (strict mode) — enforces code style and quality
4. **Build** — ensures the project compiles
5. **Tests with coverage** — runs the full test suite
6. **Coverage threshold** — verifies staged files meet 75% line coverage

All steps produce visible output on failure so issues can be diagnosed without re-running manually.

## Code Quality Limits

SwiftLint enforces the following complexity thresholds (warnings become errors in strict mode):

| Rule | Limit |
|------|-------|
| Function body length | 50 lines |
| File length | 500 lines |
| Type body length | 300 lines |
| Line length | 140 characters |
| Function parameters | 6 |
| Nesting depth | 3 levels |

## Available Commands

Run `make help` to see all available commands:

| Command | Description |
|---------|-------------|
| `make setup` | Install dependencies and configure the project |
| `make build` | Build the project |
| `make test` | Run tests |
| `make test-coverage` | Run tests with code coverage |
| `make check-coverage` | Check coverage thresholds |
| `make lint` | Run SwiftLint |
| `make lint-fix` | Run SwiftLint with autocorrect |
| `make security-scan` | Run SAST and secret scanning |
| `make open` | Open in Xcode |
| `make generate` | Regenerate Xcode project |
| `make clean` | Clean build artifacts |
